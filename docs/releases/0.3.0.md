# 0.3.0 — Open-source beta

Codebase Intelligence 0.3.0 turns the project into a stranger-ready local beta. The core workflow
is now explicit: add a repository, ask a question, and open cited source.

## Highlights

- `make demo` is the one-command local start and needs no model-provider credentials.
- `codebase-intelligence demo` starts the API and UI together; Ctrl+C stops both.
- Deterministic embeddings and extractive answers are the safe local defaults.
- The interface is focused on **Ask**, **Source**, and **Repository**.
- Example questions prefill the form instead of starting a request.
- Newly added or selected repositories return to Ask instead of retaining maintenance context.
- Maintenance and destructive actions are contained in a collapsed Repository section.
- Getting-started, extension, release, support, security, and contribution guides now form a public
  documentation path.
- Installable artifacts and the command-line surface are checked in CI.

## Upgrading from 0.2.x

Run `uv sync --frozen --all-groups` after pulling the release. Existing indexes created with a
different embedding fingerprint may require reindexing. Paid providers remain available, but they
must now be selected explicitly in `.env`.

## Validation boundary

The release is validated as a local, single-user beta. The final test, package, container, and
rendered-browser evidence is recorded before tagging. This does not claim a hosted production
deployment, public multi-tenant isolation, or provider-policy approval.

## Known limitations

- Provider and language extensions are in-tree Python changes; third-party entry-point plugins are
  not supported yet.
- The built-in deterministic retrieval mode favors reproducibility over semantic quality.
- Imported repositories are treated as untrusted and never executed, but operators must still
  review the [threat model](../security/threat-model.md) before exposing the service beyond a local
  machine.

See [Getting Started](../getting-started.md) for the first run and
